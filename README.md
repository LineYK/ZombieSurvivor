# 🧟 Zombie Survivor

> Unity 기반 3D 탑다운 좀비 서바이벌 게임

![screenshot](./img/screenshot.png)

밀려오는 좀비 웨이브에 맞서 사격, 아이템 수집, 생존 전략을 펼치는 탑다운 슈팅 서바이벌 게임입니다.  
플레이어는 마우스로 조준하고 키보드로 이동하며, 웨이브가 진행될수록 강해지는 좀비들로부터 최대한 오래 살아남아야 합니다.

## ✨ 주요 기능

### 전투 시스템
- **레이캐스트 기반 사격** — 물리 연산 없이 정밀한 즉발 히트스캔 방식 구현
- **탄약 관리** — 잔탄 · 탄창 · 예비 탄약의 3단계 탄약 시스템
- **자동/반자동 사격** — GunData의 `timeBetFire` 값에 따른 사격 모드 전환
- **시각 효과** — 총구 화염, 탄피 배출, 피격 파티클, LineRenderer 탄도 표시

### AI 시스템
- **NavMesh 기반 경로 탐색** — NavMeshAgent를 활용한 플레이어 추적
- **배회(Wander) 행동** — 타겟 미발견 시 `NavMesh.SamplePosition`을 이용한 랜덤 이동
- **공격 쿨다운** — `timeBetAttack` 기반의 근접 공격 주기 관리
- **다양한 좀비 타입** — ScriptableObject로 정의된 Default / Fast / Heavy 3종

### 웨이브 & 스폰 시스템
- **점진적 난이도 상승** — 웨이브당 좀비 수 = `wave × 1.5`
- **이벤트 기반 웨이브 전환** — `onDeath` 이벤트 구독으로 생존 좀비 수 추적
- **랜덤 스폰 포인트** — 복수의 스폰 위치에서 무작위 배치

### 아이템 시스템
- **체력팩** — 체력 회복
- **탄약팩** — 예비 탄약 보충
- **코인** — 점수 획득
- **NavMesh 기반 랜덤 스폰** — 이동 가능 영역 내 자동 배치

## 🧩 핵심 설계 패턴

### 1. 싱글톤 패턴 (Singleton)
`GameManager`와 `UIManager`는 싱글톤으로 구현되어 전역 접근 가능한 단일 인스턴스를 보장합니다.
```csharp
// GameManager.cs
public static GameManager instance { get; private set; }
void Awake() => instance = this;
```

### 2. 이벤트 기반 통신 (Event-Driven)
C#의 `Action` 이벤트를 활용하여 오브젝트 간 **느슨한 결합(Loose Coupling)** 을 유지합니다.
```csharp
// LivingEntity.cs
public event Action onDeath;      // 사망 시 구독자 알림

// ZombieSpawner.cs
zombie.onDeath += () => enemyList.Remove(zombie);  // 이벤트 구독
```

### 3. 인터페이스 추상화 (Interface Abstraction)
- **`IDamageable`** — 피해를 받을 수 있는 모든 엔티티에 대한 통일된 인터페이스
- **`IItem`** — 수집 가능한 아이템의 공통 행동 정의

→ 새로운 엔티티/아이템 추가 시 기존 코드 수정 없이 **OCP(개방-폐쇄 원칙)** 를 준수합니다.

### 4. 데이터 주도 설계 (Data-Driven Design)
`ScriptableObject`를 활용하여 게임 데이터를 코드와 분리합니다.
- **`GunData`** — 데미지, 탄창 크기, 연사 속도, 재장전 시간
- **`ZombieData`** — 체력, 데미지, 이동 속도, 스킨 컬러


### 5. 상태 머신 (State Machine)
`Gun`의 동작 상태를 `enum State { Ready, Empty, Reloading }`으로 관리하여, 사격·재장전·탄약 소진 간 전환을 명확하게 제어합니다.

### 6. 컴포넌트 기반 아키텍처 (Component-Based)
Unity의 `MonoBehaviour` 조합을 적극 활용하여 각 기능을 독립적인 컴포넌트로 분리합니다.
- **플레이어:** `PlayerInput` → `PlayerMovement` → `PlayerShooter` → `PlayerHealth`
- **적:** `Zombie` (NavMesh AI) + `ZombieSpawner` (웨이브 관리)

## 💡 개발 포인트

### 교재를 통해 학습한 내용
- 상속(`LivingEntity`)과 인터페이스(`IDamageable`, `IItem`)를 활용한 엔티티 계층 구조
- `ScriptableObject` 기반 데이터 주도 설계 (`GunData`, `ZombieData`)
- C# `Action` 이벤트를 활용한 옵저버 패턴과 느슨한 결합 구조
- 레이캐스트 + 코루틴 조합의 사격 시스템, NavMesh AI, 웨이브 스폰 시스템

### 직접 수정한 부분 — 플레이어 조작 방식 개선

교재의 기존 조작 방식은 **탱크 컨트롤(Tank Control)** 이었습니다:

| | 기존 (교재) | 변경 후 (직접 구현) |
|---|---|---|
| **W / S** | 캐릭터 기준 앞·뒤 이동 | 카메라 기준 상·하 이동 |
| **A / D** | 캐릭터 제자리 회전 | 카메라 기준 좌·우 이동 |
| **시선 방향** | 이동 방향과 동일 | 마우스 위치로 독립 조준 |

**변경 이유:** 탑다운 시점에서 탱크 컨트롤은 직관적이지 않다고 판단했습니다. 플레이어가 화면을 보며 "위쪽으로 가고 싶으면 W"라고 느끼는 것이 자연스럽고, 이동과 조준을 분리해야 사격하면서 후퇴하는 등의 전투 기동이 가능하기 때문입니다.

**구현 방식:**
- `Camera.main`의 forward/right 벡터를 XZ 평면에 투영하여 **카메라 방향 기준 이동** 구현
- 마우스 위치를 Ground 레이어에 Raycast하여 **이동과 독립된 시선 회전** 구현
- 결과적으로 이동(WASD) + 조준(마우스)이 분리된 **트윈스틱 스타일** 조작 완성

## 실행 방법

1. GitHub Releases 페이지에서 최신 릴리스를 다운로드합니다.
2. 업로드된 `.zip` 파일을 원하는 폴더에 압축 해제합니다.
3. 압축 해제한 폴더에서 `.exe` 파일을 실행합니다.
